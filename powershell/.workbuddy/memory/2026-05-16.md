## 创建 dev\vscode_ssh_remote.ps1

**功能**：
1. 读取 VS Code Remote-SSH 扩展的 SSH 配置文件（从 VS Code settings.json 的 `remote.SSH.configFile` 或默认 `~/.ssh/config`）
2. 提取所有 `IdentityFile` 路径并去重
3. 检查每个密钥文件的 NTFS 权限
4. 如果权限过于宽松（存在 Everyone/Users/Authenticated Users 访问权限），则修复为仅当前用户和 SYSTEM 可访问

**注意**：`Set-StrictMode -Version Latest` 下，单个元素的数组访问 `.Count` 会报错，需用 `@($array).Count` 包装。

## 修复 dev\encoding.ps1 无法切换编码

**问题**：脚本只设置了 `$OutputEncoding`，实际上编码有 4 个层面，缺一不可。

**根因**：
1. `$OutputEncoding` — 管道/stdout 编码
2. `[Console]::OutputEncoding` — 控制台输出编码（决定 Write-Host 显示）
3. `$PSDefaultParameterValues['*:Encoding']` — Out-File/Set-Content 等 cmdlet 默认编码
4. `chcp 65001` — 系统活跃代码页（影响子进程 git/python 等）

**修复**：重构为 `Show-Encoding`、`Set-UTF8Encoding`（含 `-NoBOM` 开关）、`Set-GBKEncoding` 三个函数，同时设置以上 4 个层面，脚本末尾默认调用 `Set-UTF8Encoding`。

## 修复 Invoke-Steps 变量展开显示

**问题**：`Invoke-Steps { Get-DnsClientCache | findstr $keyword }` 打印 `[RUN]` 时显示原始变量名 `$keyword`，而非实际值。

**修复**：在 `Lib.ps1` 的 `Invoke-Steps` 中，打印前用正则匹配脚本块中的 `$variableName`，通过 `Get-Variable` 获取当前作用域值并替换到输出字符串中。
