# 2026-05-15

## 修复 system\host_info.ps1 管道错误

**问题**：`Invoke-Steps` 将 here-string 按行逐行拆分后逐条传给 `Invoke-Expression`，多行管道命令被截断，报 "An empty pipe element is not allowed"。

**修复**：将 `host_info.ps1` 中的多行管道命令合并为单行：
```powershell
Get-NetIPAddress -AddressFamily IPv4 -PrefixOrigin Dhcp | Select-Object InterfaceAlias, IPAddress, ValidLifetime | Format-Table -AutoSize
```

**经验**：`Invoke-Steps` 不支持跨行管道，传入的 here-string 中每条命令必须在一行内写完。

## 修复 nw\check_wan.ps1 管道错误

同样的问题，多行管道被截断。改为单行字符串传入。

## 修复 nw\check_wan.ps1 Invoke-Command 参数集错误

`Invoke-Command` 是远程执行 cmdlet，字符串参数会和参数集冲突。改为 `Invoke-Step`（本地执行）。

## 修复 nw\firewall_check.ps1 输出顺序错乱

第 13-15 行直接执行管道命令，输出跑到了第 17 行 `Invoke-Step` 的 `[RUN]` 标签之前。改为统一用 `Invoke-Step` 执行，同时去掉了重复命令。

## 全局替换 Invoke-Step 为 Invoke-Steps {}

**改动范围**：Lib.ps1 + 所有调用脚本

**Lib.ps1 变更**：
- 删除 `Invoke-Step` 函数（字符串参数 + `Invoke-Expression`）
- 删除 `Invoke-Steps` 的字符串/here-string 重载（按行拆分）
- 保留唯一的 `Invoke-Steps`：接收 `[ScriptBlock]`，打印 `[RUN]` 后 `& $ScriptBlock`
- `Invoke-StepWithConfirm` 内部改为 `Invoke-Steps { Invoke-Expression $Command }`

**所有脚本变更**：
- `Invoke-Step 'string'` → `Invoke-Steps { ... }`
- `Invoke-Steps @'...'@` → `Invoke-Steps { ... }`（多行脚本块）

**涉及文件**：
- disk.ps1
- hyper-v/check.ps1, enable_hyper-v.ps1, get_cmds.ps1
- nw/dns_check.ps1, firewall_check.ps1, if_check.ps1, port_check.ps1, reset_net.ps1, route_switch.ps1, wan_check.ps1
- openssh_client.ps1, openssh_server.ps1
- system/check_env.ps1, host_info.ps1, setup.ps1
- top.ps1

**好处**：
1. 不再有多行管道被截断的问题
2. 变量在脚本块内自然展开，不需要复杂的引号转义
3. 统一调用风格，全部用 `{}` 脚本块
